다형성

Posted on 2021-08-26 In JAVA

다형성이란?

다형성(polymorphism)이란 하나의 객체가 여러 가지 타입을 가질 수 있는 것을 의미한다.

  • 하나의 코드가 여러 자료형으로 구현되어 실행되는 것
  • 같은 코드에서 여러 다른 실행 결과가 나옴
  • 정보은닉, 상속과 더불어 객체지향 프로그래밍의 가장 큰 특징 중 하나임
  • 다형성을 잘 활용하면 유연하고 확장성있고, 유지보수가 편리한 프로그램을 만들수 있음
    자바에서는 다형성을 부모 클래스 타입의 참조 변수로 자식 클래스 타입의 인스터느를 참조할 수 있도록 구현하고 있다.

예제

부모 클래스 타입의 참조 변수로 자식 클래스 타입의 인스턴스를 참조할 수 있다. 하지만 그 반대로 자식 클래스의 참조 변수로 부모 클래스 타입의 인스턴스를 참조 할 수 없다. 그 이유는 자식 클래스가 부모 클래스보다 멤버의 개수가 더 많기 때문이다.

또한 이런식으로 사용했을 경우 pc는 부모 클래스의 메소드만 접근할 수 있다.

다형성을 사용하는 이유

그렇다면 다형성을 사용하는 이유는 무엇일까?

  • 하위의 클래스를 계속 추가 하여야 하는 경우
  • 상속과 메서드 재정의를 활용하여 확장성 있는 프로그램을 만들 수 있음
  • 코드의 유지보수가 쉬움
  • 상위 클래스에서는 공통적인 부분을 제공하고 하위 클래스에서 각 클래스에 맞는 기능 구현
  • 여러 클래스를 하나의 상위 클래스 타입으로 핸들링 가능

insteadof 연산자

insteadof 연산자는 실제로 참조하고 있는 인스턴스의 타입을 확인하고 true / faulse로 반환해 준다.
