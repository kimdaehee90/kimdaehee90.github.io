Posted on 2021-08-06 In Operating System

1990년대

  • 응용 프로그램 시대
  • GUI 환경
  • 개인용 컴퓨터 사용자의 증대
  • Windows OS 대중화
  • 네트워크 기술 발전: www 대중화
  • 오픈소스 운동의 시작:UNIX계열 OS
  • 응용 프로그램 자체 개발 및 소스 오픈

    Killer application

  • 하드웨어를 사야할 이유가 될 정도의 응용 프로그램

    2000년대 이후

  • LINUX 운영체제 등 오픈 소스 활성화
  • 가상 머신 등 활성화

    응용 프로그램, 운영체제 컴퓨터 하드웨어의 관계

  • 운영체제는 응용 프로그램이 요청하는 메모리를 허가 분배 및 cpu 시간을 제공
  • 운영체제는 응용 프로그램이 요청하는 io device 사용을 허가/제어

운영체제는 사용자 인터페이스 제공

  • Shell: 사용자가 운영체제 기능과 서비스를 조작할 수 있도록 인터페이스 제공하는 프로그램으로 CLI/GUI가 존재

    운영체제는 응용프로램을 위해 인터페이스 제공

  • API

  • C library

  • 시스템 콜: 운영체제가 각 기능을 사용할 수 있도록 명령 또는 함수 제공

  • API 내부에는 시스템콜을 호출하는 형태로 대부분 만들어짐

    CPU protection rings

  • 사용자 모드: 응용 프로그램이 사용

  • 커널모드: os가 사용

    멀티 태스킹과 멀티 프로세싱

  • 멀티태스킹: 단일cpu

  • 멀티 프로세싱: 여러 cpu에 하나의 프로그램을 병렬로 실행해 속도를 극대화

    멀티 프로그래밍

  • Cpu를 최대한 많이 쓰도록 활용도를 높이는것

  • 시분할 시스템, 멀티 프로그래밍, 멀티 태스킹이 유사한 의미로 통용

  • Ws = wb.create_sheet(‘sheet_test2): sheet_test2 시트를 만드는 것

    프로세스

  • 메모리에 올려져 실행중인 프로그램

  • Task, job과 혼용

  • 응용 프로그램은 하나의 프로세스로 이뤄질수도 있고 여러 프로세스로 이뤄질수도 있음

    스케쥴러 및 스케쥴링 알고리즘

  • 프로세스를 실행시킴

  • Fifo 스케쥴러: 먼저 들어간게 먼저 나오는 형태

  • Sjf: 가장 프로세스 실행시간이 짧은 프로세스부터 실행

  • 우선순위 기반 스케쥴러: 정적 우선순위/동적 우선순위

  • Round robin: 순서대로 들어가서 실행 후 최 후순위로 돌아가 다시 순서에 맞춰 실행을 반복
